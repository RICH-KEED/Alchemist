# Pre-Ship Performance Checklist

> Run before every release build. All items must be checked.
> Generated by Performance Profiler (#39). House style: [`../../references/CONVENTIONS.md`](../../references/CONVENTIONS.md).

---

## 1. Const audit

- [ ] Run `dart analyze --fatal-infos` — zero "prefer_const_constructors" or "prefer_const_literals" infos.
  ```bash
  flutter analyze 2>&1 | grep -i "const"
  ```

- [ ] Search for `new <Widget>(` — no widget constructors without `const` in the tree.
  ```bash
  grep -r "new [A-Z]" lib/ --include="*.dart" | grep -v "// "
  ```

- [ ] Verify all `StatelessWidget` constructors are `const`.

---

## 2. Rebuild audit

- [ ] Enable `debugProfileBuilds` in `main.dart` during a profiling run; no widget exceeds 50 rebuilds per frame.
  ```dart
  // In main.dart, temporarily:
  void main() {
    debugProfileBuilds = true;
    runApp(const App());
  }
  ```
  Then in DevTools > Performance > Frame rendering stats — inspect the per-frame rebuild table.

- [ ] Audit every `ref.watch(provider)` in presentation code — none watch the whole object when only one field is used. Replace with `ref.watch(provider.select((p) => p.field))`.
  ```bash
  grep -r "ref.watch(" lib/ --include="*.dart" | grep -v "\.select"
  ```

- [ ] Verify no `setState` calls rebuild more than ~3 widgets (check call-site tree depth).

- [ ] Remove all `print`, `debugPrint`, and `toString` calls in production build paths.
  ```bash
  grep -r "print\|debugPrint" lib/ --include="*.dart" | grep -v "test/"
  ```

---

## 3. RepaintBoundary audit

- [ ] Scrolling lists: `ListView.builder` / `GridView.builder` in use (not `Column` + `SingleChildScrollView`).
  ```bash
  grep -r "SingleChildScrollView.*Column\|Column.*SingleChildScrollView" lib/ --include="*.dart"
  ```
  If any match, evaluate whether the child count warrants a builder.

- [ ] Lists with fixed item sizes have `itemExtent` or `prototypeItem` set.
  ```bash
  grep -r "ListView.builder" lib/ --include="*.dart" | grep -v "itemExtent\|prototypeItem"
  ```

- [ ] Complex static subtrees (animated charts, heavy custom paints, maps) are wrapped in `RepaintBoundary`.
  ```bash
  grep -r "CustomPaint\|CustomPainter" lib/ --include="*.dart"
  ```
  Each usage should have a `RepaintBoundary` ancestor when the painter is expensive or re-composited often.

- [ ] No `Opacity(opacity: 0.0)` widgets in the tree — replace with `Visibility` or conditional `if`.
  ```bash
  grep -r "Opacity(opacity: 0.0)" lib/ --include="*.dart"
  ```

- [ ] No `BackdropFilter` inside scrolling list items — extract above the list.

---

## 4. List builder audit

- [ ] All large lists (> 20 items) use `ListView.builder` or `GridView.builder` with a stable `Key`.
- [ ] `itemExtent` set on all lists with uniform item sizes.
- [ ] No unbounded `ListView` inside another scrollable without `shrinkWrap: true` (or, better, restructured as a `SliverList`).
  ```bash
  grep -r "shrinkWrap: true" lib/ --include="*.dart"
  ```
  Every `shrinkWrap` usage is a candidate for `SliverList` refactoring.

---

## 5. Image cache audit

- [ ] All `Image.network` calls use `cacheWidth` / `cacheHeight` to bound decode resolution.
  ```bash
  grep -r "Image.network" lib/ --include="*.dart" | grep -v "cacheWidth\|cacheHeight"
  ```

- [ ] `precacheImage` called for images that appear on first frame (splash, hero images, onboarding).
  ```bash
  grep -r "precacheImage" lib/ --include="*.dart"
  ```

- [ ] Image cache size tuned via `PaintingBinding.instance.imageCache.maximumSize` if the app loads 30+ images.

---

## 6. Shader warm-up

- [ ] SkSL captured by exercising every screen and transition in profile mode.
  ```bash
  flutter run --profile --cache-sksl --write-sksl-on-exit=sksl_capture.json
  ```

- [ ] SkSL bundled in release builds.
  ```bash
  flutter build apk --bundle-sksl-path=sksl_capture.json
  flutter build appbundle --bundle-sksl-path=sksl_capture.json
  ```

- [ ] CI pipeline re-captures SkSL when UI changes (or at least validates the existing bundle is up to date).
  ```bash
  flutter test --update-sksl
  ```

- [ ] Verify zero `GrGLGpu::compileShader` events in a fresh-install-first-launch trace.

---

## 7. Isolate offload check

- [ ] No JSON parsing of > 50 KB payloads on the UI thread — use `Isolate.run` or `compute()`.
  ```bash
  grep -r "jsonDecode\|json.decode" lib/ --include="*.dart"
  ```

- [ ] No sorting/filtering of > 500 items in build or animation callbacks.

- [ ] Crypto/hashing operations offloaded to an isolate.

- [ ] Image processing (compression, resize, format conversion) offloaded to an isolate.

---

## 8. Memory check

- [ ] Memory snapshot taken after a full navigation flow — no unbounded growth.
  In DevTools > Memory: snapshot before and after, compare. Look for leaks in `BuildContext` or disposed controllers.

- [ ] All `TextEditingController`, `ScrollController`, `AnimationController`, and `FocusNode` instances disposed in `dispose()`.
  ```bash
  grep -r "Controller\|FocusNode\|TextEditingController\|ScrollController" lib/ --include="*.dart" | grep -v "dispose\|// "
  ```

- [ ] Image cache eviction strategy tested (navigate a heavy image feed for 60 seconds — memory should plateau, not climb).

- [ ] No `setState` or `ref.read` calls after `dispose()` (enabled by mounted checks or provider-scoped listeners).

---

## 9. Final profile pass

- [ ] Run the app with `flutter run --profile` and navigate the full MVP flow while recording.
- [ ] Export the timeline: DevTools > Performance > **Export**.
- [ ] Inspect: zero frames exceed 16.67 ms in the steady state (first-run shader frames are allowed only if SkSL is bundled and verified).
- [ ] Raster thread time < 8 ms on 95th percentile frames.
- [ ] UI thread time < 8 ms on 95th percentile frames.
- [ ] Rebuild counts < 50 per frame in scrolling screens.
