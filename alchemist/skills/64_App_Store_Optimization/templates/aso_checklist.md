# ASO Checklist — Google Play Store

## 1. Title (30 char max)

- [ ] Primary keyword in the first 15 characters
- [ ] Brand name included (if space allows; otherwise brand goes in developer name)
- [ ] No keyword repetition
- [ ] Readable to a human (not "KW1 KW2 KW3 App")
- [ ] Distinct from top 5 competitors' titles

## 2. Short Description (80 char max)

- [ ] Starts with a verb or benefit statement
- [ ] Contains 2-3 secondary keywords naturally
- [ ] Includes a differentiator ("offline", "AI-powered", "privacy-first")
- [ ] Ends with a period (polish)

## 3. Full Description (4000 char max)

- [ ] First 2-3 sentences: primary keyword 1x + secondary keyword 2-3x
- [ ] Primary keyword appears 3-5x total (natural, not stuffed)
- [ ] Each secondary keyword appears 1-2x
- [ ] Short paragraphs: 2-3 sentences max. Each paragraph a single idea.
- [ ] Feature list in bullet format, benefit-first ("Track expenses in 3 taps" not "Expense tracking feature")
- [ ] Social proof placeholder (or real quote if available)
- [ ] Trust signals: privacy, offline, no-ads, open-source, data-export
- [ ] Call to action in the last paragraph
- [ ] No competitor names mentioned (against Play Store policy)
- [ ] No excessive CAPS or exclamation marks!!!
- [ ] Contact/support info at the bottom

## 4. Icon (512x512 adaptive)

- [ ] Recognizable at 48dp (notification bar size) — no fine detail
- [ ] Distinct silhouette from top competitors
- [ ] No text in the icon (text at icon size is illegible)
- [ ] Adaptive icon: foreground layer + background layer (Android 8+)
- [ ] Legacy icon rendered correctly (round/square/squircle masks)

## 5. Feature Graphic (1024x500)

- [ ] Brand name visible
- [ ] Tagline or value prop
- [ ] No critical info near edges (may be cropped)
- [ ] Reads well at thumbnail size (list page)

## 6. Screenshots (JPEG/PNG, up to 8, min 320px, max 3840px)

- [ ] Ordered by importance (not by app flow)
- [ ] Slot 1: core value prop — best feature, not onboarding
- [ ] Slots 2-4: key features, one per screenshot
- [ ] Slots 5-7: differentiators
- [ ] Slot 8: social proof / review highlight
- [ ] Each screenshot has a caption overlay (text on the image)
- [ ] Captions are benefit-focused ("Scan receipts in one tap") not feature-names ("Receipt Scanner")
- [ ] No device bezel/chrome in first 2 screenshots
- [ ] Light + dark variants available (at minimum for slots 1-2)
- [ ] Caption text contrast ≥ 4.5:1 against the screenshot background
- [ ] No personal data visible in screenshots (fake data only)
- [ ] No notch/cutout renders on screenshots

## 7. Localization (per locale)

- [ ] Title + short description + full description translated
- [ ] Keywords researched per locale (not direct translations)
- [ ] Screenshot captions localized (separate image files or dynamic overlay)
- [ ] Cultural fit reviewed: colors, imagery, RTL text direction if applicable

## 8. Rating & Review Strategy

- [ ] In-app rating prompt timed correctly (after a positive action, not on first launch)
- [ ] Review response templates prepared for common feedback
- [ ] Not prompting users who already rated (save a flag)

## 9. Post-launch Monitoring

- [ ] Google Play Console acquisition reports → keyword impressions, visit rate, install rate
- [ ] A/B test screenshots via Play Console experiments (if available in region)
- [ ] Watch competitor listing changes
- [ ] Re-run ASO audit every 3 months or after major feature release
