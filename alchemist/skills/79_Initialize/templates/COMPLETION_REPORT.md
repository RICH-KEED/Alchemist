## 🚨 Manual steps — human action required

These cannot be automated and must be completed before Play Store submission.

### 1. Generate upload keystore
```
keytool -genkey -v -keystore ~/upload-keystore.jks \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -alias upload
```
Store the passwords securely. You'll need them for every release.

### 2. Enroll in Play App Signing
1. Go to [Play Console](https://play.google.com/console) → your app → Setup → App signing
2. Upload the PEPK/JAR from `keytool` to enroll your upload key
3. The app signing key (managed by Google) is what users install — your upload key is only for submitting updates

### 3. SHA-256 fingerprints for Firebase / Google services
```
keytool -list -v -keystore ~/upload-keystore.jks -alias upload
# Copy the SHA-256: line
```
Add this fingerprint to:
- [ ] Firebase Console → Project Settings → SHA certificate fingerprints
- [ ] Google Cloud Console → APIs & Services → Credentials (if using OAuth)

### 4. Download `google-services.json`
1. Go to [Firebase Console](https://console.firebase.google.com) → your project → Project Settings
2. Add your Android app (package name from `android/app/build.gradle`)
3. Download `google-services.json` → place in `android/app/`
4. ⚠️ This file is gitignored — never commit it

### 5. Create Play Console listing
1. [Play Console](https://play.google.com/console) → Create app
2. Fill in:
   - **App name, short description, full description** → copy from `publish/play_*.txt`
   - **Feature graphic** → upload `publish/play_feature_graphic.png`
   - **Screenshots** → capture via `flutter run --release` on a real device (Play requires real screenshots)
   - **App icon** → upload `publish/play_icon.png` (512×512, 32-bit PNG with alpha)

### 6. Data Safety form
1. Play Console → App content → Data safety
2. Review the answers in `publish/` (generated by skill 59 Privacy scan)
3. Verify against your actual data practices — **you are legally responsible for accuracy**

### 7. Privacy policy URL
1. Host `publish/privacy_policy.md` at a public URL
2. Add the URL to Play Console → App content → Privacy policy

### 8. Content rating questionnaire
1. Play Console → App content → Content rating
2. Complete the questionnaire — takes ~5 minutes

### 9. OAuth client secrets (if using Google Sign-In / Firebase Auth)
1. [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials
2. Configure OAuth consent screen + create OAuth client IDs
3. Note: SHA-256 from step 3 must already be registered

### 10. Apple Developer (if iOS)
1. [Apple Developer](https://developer.apple.com) — paid account required ($99/year)
2. Create App ID + provisioning profile
3. App Store Connect → create app, upload via Xcode/Transporter
